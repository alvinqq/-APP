
# 项目核心规则 (Project Rules)

## 基础规约
- 所有代码变更必须同步考虑文档更新。
- 代码注释请使用中文，确保清晰易懂。

## 🚀 自动文档生成规则 (核心!)
每次完成代码修改后，你必须自动执行以下文档更新操作：
1.  **创建总结文档**：在 `docs/summaries/` 目录下，按日期创建子目录（格式：`YYYY-MM-DD`），并在其中生成一个 Markdown 文件。
2.  **文件命名**：文档命名格式为 `HH-MM-SS_feature_summary.md`（例如：`15-30-22_user_login_feature_summary.md`）。
3.  **文档内容必须包含**：
    - **本次修改的功能点**：用一句话概括。
    - **涉及的文件列表**：列出所有被修改、新增或删除的文件路径。
    - **主要代码逻辑说明**：简要描述核心的代码改动和业务逻辑。
    - **API 变更记录**：如果有接口变更，请详细记录。

## 代码质量要求
- 遵循项目现有代码风格和命名规范。
- 每个公开的函数或方法必须有注释说明其用途。
