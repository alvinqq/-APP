# 功能修改总结

## 本次修改的功能点
1. 为异常预警和待办任务模块添加"查看更多"功能，默认显示两条记录，超出部分可点击查看全部
2. 创建异常预警列表页面，展示所有异常预警详情和处理流程
3. 创建待办任务列表页面，展示所有待办任务详情
4. 整合加盟商首页的门店核心指标和销售报表模块，创建新的"门店经营数据"模块
5. 优化首页模块布局，确保一屏展示更多信息

## 涉及的文件列表
- `App.tsx`：更新页面状态管理、添加新页面路由逻辑、整合加盟商首页模块
- `components/ExceptionAlertListPage.tsx`：新增异常预警列表页面组件
- `components/TaskListPage.tsx`：新增待办任务列表页面组件
- `components/ExceptionAlertModule.tsx`：修改异常预警模块，添加查看更多功能

## 主要代码逻辑说明
1. **异常预警模块**：修改了 `ExceptionAlertModule` 组件，添加了 `onViewMore` 回调，当预警数量超过2条时显示"查看更多"按钮
2. **待办任务模块**：在 App.tsx 中修改了待办任务展示逻辑，添加了"查看更多"按钮，点击后跳转到任务列表页面
3. **页面路由**：在 App.tsx 中添加了 `currentPage` 状态，用于管理异常预警列表和待办任务列表页面的显示
4. **加盟商首页整合**：将"销售报表"和"门店核心指标"两个模块合并为"门店经营数据"模块，保持原有功能不变

## API 变更记录
- 新增 `handleViewMoreAlerts` 函数：处理异常预警查看更多逻辑
- 新增 `handleAlertBack` 函数：处理异常预警列表页面返回逻辑
- 新增 `handleTaskBack` 函数：处理待办任务列表页面返回逻辑
- 更新 `ExceptionAlertModule` 组件 props：添加 `onViewMore` 参数

## 功能效果
1. 异常预警和待办任务模块默认只显示2条记录，减少首页信息过载
2. 点击"查看更多"按钮可进入对应的列表页面，查看完整的预警和任务信息
3. 加盟商首页的"门店经营数据"模块整合了销售报表和核心指标，使信息更集中
4. 整体布局更加紧凑，一屏可展示更多模块信息
