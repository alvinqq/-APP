# 绝知 APP 功能架构补充说明

> 文档创建时间：2026-03-03  
> 作者：AI 产品搭档  
> 版本：v1.0

## 概述

本文档基于《绝知 app 产品架构.md》的功能架构描述，补充实现了项目中缺失的核心功能模块。

## 已实现功能模块

### 1. 工作台（金刚区）✅

**文件路径**: `/components/Workbench.tsx`

**功能描述**:
- 整合所有常用功能模块的快捷入口
- 支持按类别组织：常用功能、销售类、店务类、系统功能
- 支持收藏/取消收藏功能（星标标记）
- 底部导航栏支持页面切换（首页、工作台、数据、我的）

**包含的功能项**:
| 类别 | 功能项 |
|------|--------|
| 常用功能 | 扫码开单、库存查询、订单管理、销售报表 |
| 销售类 | POS 登录、扫码核券、临期促销、移动收银 |
| 店务类 | 支付进件、维修申报、来访报备、IT 服务 |
| 系统功能 | 绝智、在线运营、授信、企业微信 |

**技术实现**:
- React 函数组件
- 支持功能项收藏状态管理
- 响应式网格布局（4 列）
- 底部导航栏集成

---

### 2. 店务管理模块 ✅

**文件路径**: `/components/StoreAffairsPage.tsx`

**功能描述**:
- 统一管理门店各类事务申请
- 支持查看事项状态（待处理、处理中、已完成）
- 显示申请流程说明
- 提供快速提交申请入口

**包含的子功能**:
1. **支付进件**: 商户入驻、支付渠道开通申请
2. **维修申报**: 设备故障报修、维护申请
3. **来访报备**: 外部人员来访登记、审批
4. **IT 服务**: 系统问题、网络故障、硬件支持

**页面结构**:
- 概览卡片：显示待处理、处理中、已完成数量
- 事项列表：展示所有店务事项及状态
- 详情页：显示申请流程、提交表单

**技术实现**:
- 状态管理：pending、processing、completed
- 流程步骤展示（1-2-3 步）
- 时间戳显示
- 状态徽章（不同颜色区分）

---

### 3. 进销存管理模块 ✅

**文件路径**: `/components/InventoryManagementPage.tsx`

**功能描述**:
- 完整的库存管理功能集合
- 支持概览和明细两种视图切换
- 库存预警提示
- 快捷操作入口

**包含的子功能**:
| 功能 | 描述 | 颜色标识 |
|------|------|----------|
| 报收投 | 商品报货、收货、退货管理 | 蓝色 |
| 调拨 | 店间商品调拨、转移 | 紫色 |
| 试吃 | 试吃品登记、核销 | 橙色 |
| 报损 | 商品损耗登记、审批 | 红色 |
| 库存盘点 | 定期盘点、库存核对 | 绿色 |
| 门店拆包 | 一物一码溯源管理 | 靛蓝色 |

**核心指标**:
- 总 SKU 数
- 库存总额
- 临期商品数
- 库存预警数

**库存状态**:
- 正常（绿色）
- 库存不足（橙色）
- 临期（红色）

**技术实现**:
- Tab 切换：概览/明细
- 网格布局展示快捷操作
- 库存预警卡片（橙色警示）
- 商品列表展示

### 4. 数据看板模块 ✅

**文件路径**: `/components/DataDashboardPage.tsx`

**功能描述**:
- 经营数据全景展示
- 营收统计分析
- 钱包管理（营收钱包、货款钱包、货补钱包）
- 收款统计分析

**核心功能**:

#### 4.1 经营看板（首页）
- 今日实时营收卡片（红色渐变）
- 核心指标快速查看（营收统计、钱包余额）
- 收款统计分析（微信支付、支付宝、现金、其他）
- 关键指标趋势（同比、环比、目标达成率）

#### 4.2 营收统计分析
- 本月累计营收录入
- 四大核心指标：
  - 实收金额
  - 交易笔数
  - 客单价
  - 毛利率
- 营收趋势图（AreaChart 面积图）
- 渠道收入分析（到店、外卖、会员、其他）

#### 4.3 钱包管理
支持三个钱包切换查看：
1. **营收钱包**: 蓝色主题
   - 营收到手金额
   - 品牌分账收入
   - 三方结算金额

2. **货款钱包**: 绿色主题
   - 货款账户总余额
   - 月度货款支出/充值
   - 工厂拆分展示

3. **货补钱包**: 紫色主题
   - 可用余额
   - 本月已返/已用
   - 报货占用

**技术实现**:
- Recharts 图表库集成
- Tab 切换导航
- 响应式卡片布局
- 渐变色数据卡片
- 收支明细列表

### 5. 巡检系统模块 ✅

**文件路径**: `/components/InspectionPage.tsx`

**功能描述**:
- 完整的巡检管理流程
- 巡检任务分配与执行
- 巡检记录管理
- 异常整改上传

**核心功能**:

#### 5.1 巡检概况
- 巡检完成率统计
- 门店健康分展示
- 总巡检次数
- 红牌警告次数

#### 5.2 巡检任务管理
- 待执行巡检列表
- 任务优先级标识（紧急/重要/普通）
- 任务状态跟踪（待执行/进行中/已完成）
- 截止日期提醒

#### 5.3 巡检记录
- 历史巡检记录查询
- 巡检评分展示（90+ 绿色、80+ 橙色、80 以下红色）
- 问题项统计
- 红牌警告标记

#### 5.4 巡检详情
- 门店信息展示
- 评分详情
- 问题清单（带图标警示）
- 整改要求说明

#### 5.5 整改上传
- 现场拍照功能
- 相册上传功能
- 整改结果提交

**技术实现**:
- 多 Tab 导航（概览/任务/记录）
- 状态徽章系统
- 评分颜色编码
- 相机/上传按钮
- 问题清单卡片

## 技术栈说明

### 使用的核心技术
- **React 18**: 函数组件 + Hooks
- **TypeScript**: 类型安全
- **Tailwind CSS**: 原子化 CSS
- **Recharts**: 数据可视化图表
- **Lucide React**: 图标库

### 设计规范
- **颜色系统**:
  - 主色：红色 (#dc2626)
  - 辅助色：蓝、绿、橙、紫、靛
  - 背景色：#F8FAFC

- **布局规范**:
  - 最大宽度：430px（移动端适配）
  - 圆角：rounded-xl (12px), rounded-2xl (16px)
  - 阴影：shadow-sm, shadow-lg

- **字体规范**:
  - 标题：text-base, font-bold
  - 正文：text-sm, text-xs
  - 颜色：text-gray-800, text-gray-600, text-gray-500
